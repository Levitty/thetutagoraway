Facebook Auto-Posting Setup

One-time setup to let the Tutagora agent post to your Facebook Page

Action needed: The scheduled task is already created and will run every Monday, Wednesday, and Friday at 10am. But it needs your Facebook Page credentials to actually post. Follow the steps below — it takes about 15 minutes.

Part 1: Create a Facebook Page (if you don't have one)

1

Go to facebook.com/pages/create

Choose "Business or Brand", name it Tutagora, category: Education or Tutoring/Teaching Center.

2

Add a profile picture (Tutagora logo) and cover photo. Fill in the bio with something like: "Connecting Kenyan students with verified tutors. Live lessons, AI Math Tutor, and school partnerships. tutagora.com"

Part 2: Create a Meta Developer App

3

Go to developers.facebook.com and log in with the same account that owns the Tutagora Facebook Page.

4

Click "My Apps" (top right) → "Create App".

Select "Business" as the app type. Name it something like "Tutagora Auto-Post".

5

Once the app is created, go to the app dashboard. On the left sidebar, click "Add Product" and find "Facebook Login" → click "Set Up".

Part 3: Get Your Page Access Token

6

Go to the Graph API Explorer.

In the top-right dropdown, select your app ("Tutagora Auto-Post").

7

Click "Generate Access Token". When prompted for permissions, make sure to check:

pages_manage_posts
pages_read_engagement
pages_show_list

Click "Generate Access Token" and authorize when the Facebook popup appears.

8

Now you have a User Access Token. You need to convert it to a Page Access Token.

In the Graph API Explorer, paste this into the query field and click Submit:

me/accounts

This returns a list of Pages you manage. Find "Tutagora" in the results. Copy the access_token value and the id value (that's your Page ID).

9

Make the token long-lived (so it doesn't expire in 1 hour). In the Graph API Explorer, paste this URL:

oauth/access_token?grant_type=fb_exchange_token&client_id=YOUR_APP_ID&client_secret=YOUR_APP_SECRET&fb_exchange_token=YOUR_SHORT_LIVED_TOKEN

Replace YOUR_APP_ID and YOUR_APP_SECRET (found in your app's Settings → Basic) and YOUR_SHORT_LIVED_TOKEN with the Page token from step 8.

The response gives you a long-lived token that lasts ~60 days.

Tip: Page Access Tokens obtained through a long-lived User Access Token don't expire. So if you get a long-lived user token first, then call me/accounts, the resulting page token is permanent.

Part 4: Update Your Config File

10

Open the file facebook-config.json in your Tutagora project folder. It looks like this:

{
"page_id": "YOUR_PAGE_ID_HERE",
"access_token": "YOUR_PAGE_ACCESS_TOKEN_HERE"
}

Replace the placeholder values with your actual Page ID and Page Access Token from step 8/9:

{
"page_id": "123456789012345",
"access_token": "EAABsbCS1iHgBO0vK..."
}

Save the file. That's it — the agent will use these credentials to post.

Keep your token safe: Don't share the facebook-config.json file or commit it to GitHub. It contains credentials that give access to post on your Facebook Page.

Part 5: Test It

11

To test the agent, open Claude and ask: "Run the tutagora-facebook-posts task"

It will generate a post and attempt to publish it. If it works, you'll see a success message with the Facebook post ID. If not, it will tell you exactly what went wrong (usually an expired token or wrong permissions).

Troubleshooting

"Token expired"Generate a new token following steps 6–9
"Permission denied"Make sure pages_manage_posts is enabled
"Page not found"Double-check the page_id in facebook-config.json
Posts not appearingCheck if your app is still in Development mode — it still works for Pages you own
